# 記事コンテンツの規約（content/posts）

このファイルは「記事をどう書くか」の唯一の真実源（SoT）。frontmatter のスキーマ定義は
`src/content.config.ts`、ここはその運用ルールと文体ガイド。`lint-post` hook と `new-log`
スキルはここを参照する。

## 置き場所とファイル名

- 記事は `src/content/posts/<slug>.md` に置く（1記事=1ファイル）。
- `<slug>` は英小文字・ハイフン区切り。URL は `/posts/<slug>/` になる。
  - 例: `polite-requests.md` → `/posts/polite-requests/`
- 日付はファイル名に入れない（frontmatter の `date` が真実源。並び順もこれを使う）。

## frontmatter（必須・任意）

```yaml
---
title: "「Could you...?」で頼む — 依頼の丁寧さレベル"   # 必須
date: 2026-06-24                                      # 必須 (YYYY-MM-DD)
summary: "一覧カードと OG に出る一言要約。"              # 必須 (1〜2文)
tags: ["依頼", "丁寧さ"]                               # 任意 (既存タグを優先して再利用)
level: beginner                                       # 任意 beginner|intermediate|advanced
draft: false                                          # 任意 true で一覧・ビルドから除外
---
```

- `level` の既定は `beginner`。学習当時の体感難易度で付ける（🌱/⚙️/🚀 がカードに出る）。
- `tags` は **新語を増やす前に既存タグを確認**して揃える（`/tags/` で一覧できる）。表記ゆれを作らない。
- `date` は一覧の並び順（新しい順）を決める。**新記事を先頭に出すには既存の最新より新しい時刻**にする。
  同日に複数足すときは `2026-06-19T22:00:00` のように**時刻**まで入れて順序を安定させる。

## 粒度の基準（記事の濃さを揃える）

既存記事 `src/content/posts/*.md` が**正解の見本**。新規も同じ濃さに揃える。

- **1記事=1トピック**（例:「依頼」「あいづち」「カフェでの注文」）。題材が広いときは焦点を1つに
  絞る。素材が複数トピックなら**分割**する。
- 本文の節は **4〜6個** が目安。「濃い」と感じたら削るか分割（詰め込まない）。
- 読者は**日本語ネイティブの英語学習者**。効く箇所で「日本語の直訳だとこう言いがち／ネイティブは
  こう言う」を `> 🧭` で1行添える。

## 文体・体裁ガイド

- **冒頭1〜2文で「何の話か」を要約**してから本論に入る。
- 見出しは `##`（大）/`###`（小）。`#` は使わない（title が H1）。
- 英語表現は **太字**、直後に日本語訳を添える（例: **No worries.** — 気にしないで）。
- 会話例はコードブロック `text` で書く（```text）。A:/B: 形式で2〜6行、下に和訳か補足を添える。
- **要点ブロック**は引用 `>` を使う。絵文字を1つ添えると効く
  （`> 💡` コツ / `> ⚠️` 失礼になる・誤解される注意 / `> 🧭` 日本語感覚とのズレ / `> ⭐` 最重要）。
- 表現の比較（丁寧さレベル・ニュアンス差・場面別）は**表**にする。
- 記事末に「まとめ」（箇条書き3〜5個）と、可能なら「次にやること」を置く。
- 内部リンクは `/posts/<slug>/` `/tags/<tag>/` の**ルート絶対**で書く（base は自動付与される）。
  リンクテキストは **URL をそのまま見せず、リンク先の記事タイトル（または説明語句）にする**
  （例: `[「No way! — 驚き・感想を一言で返すリアクション表現」](/posts/reaction-phrases/)`）。
- シリーズものは冒頭に `(n/総数)` 表示、末尾に前後リンクを付ける。

## やらないこと

- `dist/` `.astro/`（生成物）は触らない（`guard-generated` hook が止める）。
- 恒久的に残したい設計判断は記事ではなく `docs/decisions/` に薄く記録する（[decisions](../../docs/README.md)）。
- 実在が怪しい表現・古すぎるスラングは載せない（自然に使われている定番だけを扱う）。
