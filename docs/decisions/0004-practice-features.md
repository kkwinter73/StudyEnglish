# 0004. 練習機能は「記事の書式からビルド時抽出」で作る

**決定**: 瞬間英作文ドリル（/drill/）・シャドーイング（/practice/）・今日のフレーズは、
専用データを持たず、**記事本文の書式（「**英語** — 日本語」と ```text 会話例）をビルド時に
機械抽出**して生成する（`src/phrases.ts`）。進捗（覚えた）は localStorage、音声はブラウザの
Web Speech API を使い、**サーバー・外部サービス・依存追加はゼロ**のまま運用する。

## なぜ

- 記事を書き足すだけで練習素材が自動で増える（データの二重管理をしない。書式が SoT）。
- 静的サイト＋GitHub Pages の構成を崩さない（DB・API・アカウント不要）。
- TTS はブラウザ内蔵（speechSynthesis）で十分。品質を上げたくなったら音声ファイル生成に差し替え可能。

## 結果・前提

- 記事の書式が抽出の前提になった。ルールは `.claude/rules/content.md` に明記（書式を崩すとドリルに載らない）。
- 「覚えた」記録は端末ローカル（localStorage）。端末をまたぐ同期はやらない（個人運用・割り切り）。
