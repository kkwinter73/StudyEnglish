# 0001. ブログ基盤に Astro を採用する

**決定**: 英語勉強ログのブログ基盤に **Astro（静的サイトジェネレータ）** を採用する。

## なぜ（コードを読んでも出てこない理由)

- 先行リポジトリ [StudyLog](https://github.com/kkwinter73/StudyLog) で構成・運用が実証済み。
  同じ作りを流用することで、記事執筆に集中できる。
- デザイン自由度が高く、「技術ブログ＋遊び心＋おしゃれ」という見た目要件を素のCSSで作り込める。
- Markdown をそのまま記事化でき、Notion からの素材移行と相性がよい（運用は都度 Markdown を整形）。
- ローカルで即プレビューでき、編集→確認の反復が速い。

## 却下した選択肢

- **Hugo / Next.js**: StudyLog の 0001 と同じ理由（デザイン作り込みの重さ／動的要件なしで過剰）。
- **ゼロから新設計**: 記事以外は StudyLog の構成を踏襲する方が学習コンテンツに時間を使える。
